# SaaS для GEO/AEO Аналитики (Воронка нулевого клика)
Проект: B2B платформа для отслеживания упоминаний брендов в ответах ИИ (ChatGPT, Perplexity, Claude) и генерации рекомендаций по улучшению AI Visibility Score.

## 1. Технологический стек
- **Фреймворк:** Next.js 15 (App Router, TypeScript).
- **Стилизация:** Tailwind CSS + shadcn/ui (базовые компоненты) + Tremor (аналитические графики и дашборды).
- **База данных:** PostgreSQL + Prisma ORM.
- **Аутентификация:** NextAuth.js (Auth.js) v5. Локальное хранение сессий и пользователей (Credentials: Email/Password).
- **Фоновые задачи:** BullMQ + Redis. Полностью своя инфраструктура очередей для обхода таймаутов при долгих запросах.
- **Парсинг:** Playwright (Open-source парсинг напрямую с сервера).
- **Внешние API:** OpenRouter API (единая точка входа для LLM: используем Perplexity Sonar для веб-поиска упоминаний и Claude 3.5 Sonnet / GPT-4o для генерации итогового JSON-отчета).

## 2. Архитектура БД (Prisma)
Система использует следующие основные модели:
- `User`: id (UUID), name, email, password (hashed), plan (FREE/PRO/AGENCY), credits (Int, по умолчанию 50).
- `Project`: id, userId (связь с User), url, name.
- `Report`: id, projectId (связь с Project), status (PROCESSING/COMPLETED/FAILED), overallScore (Float).
- `ShareOfVoice`: id, reportId (связь с Report), llmProvider (String), keyword, isMentioned (Boolean), competitors (Json).
- `Recommendation`: id, reportId (связь с Report), type (String), title, description, generatedCode (String, код для вставки на сайт).

## 3. Бизнес-логика и Конвейер генерации отчета (BullMQ Pipeline)
Отчет генерируется асинхронно. Процесс разбит на шаги внутри Worker'а:
1. **Инициация:** Клиент нажимает "Анализировать". Бэкенд создает `Report` со статусом `PROCESSING` и отправляет задачу в очередь BullMQ.
2. **Сбор данных (КРИТИЧНО):** Playwright открывает невидимый браузер и собирает текст сайта клиента. **Важно:** Скрипт должен ОБЯЗАТЕЛЬНО проверять доступность файла `URL + /llms.txt` и искать на страницах микроразметку Schema.org (в формате `application/ld+json`, особенно типы FAQPage, Article, LocalBusiness). Это главные факторы GEO-оптимизации.
3. **Опрос LLM:** Запросы через OpenRouter к online-моделям (например, Perplexity) для проверки, рекомендует ли ИИ этот сайт по целевым запросам (Share of Voice).
4. **Анализ конкурентов:** Если клиента нет в ИИ-выдаче, собираем данные конкурентов, которых порекомендовала нейросеть.
5. **Генерация стратегии:** Отправка собранных данных через OpenRouter (Claude 3.5 Sonnet или GPT-4o) для генерации строгого JSON с рекомендациями (исправить разметку, добавить контент и т.д.).
6. **Финализация:** Запись JSON-результатов в Prisma, смена статуса отчета на `COMPLETED`.

## 4. Экономика и Кредиты (Биллинг)
- Аудит стоит 10 кредитов.
- **Транзакционность:** Кредиты списываются с баланса пользователя (`User.credits`) СТРОГО в самом конце успешного выполнения задачи BullMQ (перед сменой статуса на `COMPLETED`), используя транзакции Prisma.
- **Защита баланса:** При ошибках сторонних API или парсера, задача падает, статус отчета меняется на `FAILED`, кредиты НЕ списываются. Настроить повторные попытки (retries) в BullMQ для обхода временных сетевых сбоев.

## 5. UI / UX Требования
- **Landing Page:** Темный SaaS-дизайн (bg-slate-950). Блок Hero с инпутом URL.
- **UX Ожидания (Frontend Polling):** Пока BullMQ работает в фоне (1-2 минуты), фронтенд делает поллинг БД (через SWR или React Query на роут `/api/reports/[id]/status` каждые 3 секунды). Пользователь видит Skeleton UI и динамический прогресс-бар с меняющимся текстом.
- **Дашборд:** После статуса `COMPLETED` редирект на готовый отчет. Использовать компоненты Tremor (Radar Chart, Donut Chart) для красивой визуализации данных из БД.
